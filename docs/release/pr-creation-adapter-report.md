## PR Adapter Functions

### `createPullRequest(options: CreatePROptions): Promise<GitHubPullRequest>`

Erstellt einen Pull Request aus einem existierenden Branch. Idempotent.

**Parameter:**
| Feld | Typ | Beschreibung |
|------|-----|-------------|
| `owner` | string | Repository-Besitzer |
| `repo` | string | Repository-Name |
| `title` | string | PR-Titel |
| `head` | string | Source-Branch (muss existieren) |
| `base` | string | Target-Branch (default: main) |
| `body` | string? | PR-Beschreibung (Markdown) |
| `draft` | boolean? | Als Draft erstellen |

**Rückgabe:** `GitHubPullRequest` mit id, number, title, body, state, head/base refs, htmlUrl.

### `listPullRequests(options: PRListOptions): Promise<GitHubPullRequest[]>`

Listet Pull Requests mit optionalen Filtern.

**Filter:**
- `state`: `'open' | 'closed' | 'all'` (default: `'open'`)
- `head`: Branch-Name für Head-Filter (Format: `owner:branch`)

### `listPullRequestFiles(owner, repo, prNumber): Promise<GitHubPRFile[]>`

Gibt geänderte Dateien eines PRs zurück mit Diff-Statistiken (additions, deletions, changes, status, filename).

## Idempotenz-Strategie

Bevor ein neuer PR erstellt wird, prüft `createPullRequest()`:

```
GET /repos/{owner}/{repo}/pulls?state=open&head={owner}:{head}
```

Wenn bereits ein offener PR für diesen Branch existiert, wird dieser zurückgegeben — **kein Duplikat**. Dies verhindert doppelte PRs bei wiederholten Runs.

## PR_CREATED Label-Lifecycle

Der `LABEL_LIFECYCLE`-Eintrag für `PR_CREATED`:

```typescript
PR_CREATED: {
  add: ['positron:pr-created'],
  remove: ['positron:running', 'positron:repo-sync', 'positron:research',
           'positron:testing', 'positron:blocked', 'positron:failed'],
}
```

Transition: `positron:running` → `positron:pr-created` (alle vorherigen Labels werden entfernt).

## `syncPrCreated()` GitHub-Kommentar

Der `GitHubStatusSyncService.syncPrCreated(input)` schreibt einen Kommentar mit:

```markdown
<!-- positron:run={runId};phase=PR_CREATE;kind=pr-created -->

## Pull Request Created

**Pull Request:** [#{prNumber}]({prUrl})
**Branch:** `{branchName}`
**Closes:** #{issueNumber}
```

Zusätzlich wird das `positron:pr-created`-Label gesetzt.
