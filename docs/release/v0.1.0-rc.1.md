# Positron v0.1.0-rc.1 — Release Candidate 1

**Datum:** 2026-05-26

## Überblick

Erster Release Candidate von Positron — dem Evidence-Gated GitHub Issue Execution System.
Dieser RC markiert den Abschluss der ersten Funktionsentwicklung und den Übergang zur Stabilisierung.

## Enthaltene Features

### Pipeline (Issue #1-#13, #14-#27, #28-#62)
- Vollständige 27-Phasen-Pipeline: QUEUED → CLAIMED → SPECIFY → ... → DONE/FAILED
- Speckit-Workflow: Spec → Plan → Tasks → Analyze → Review → Implement → Test → Verify
- Fix-Loop mit Expo-Backoff (max 3 Wiederholungen)
- Evidence-Gates: Jede Phase produziert prüfbare Artefakte
- Run-State-Machine mit Resumption aus Events

### GitHub Integration
- GitHub Adapter (Fake/Real Strategy Pattern)
- Issue-Listen, PR-Erstellung, Merge, Labels, Review-Request
- Auto-Merge mit Dry-Run, Kill-Switch, Mergeability-Prüfung
- GitHub Watcher für automatische Run-Erstellung

### Web UI (Issue #9, #22, #27, #52, #55)
- App Shell mit Sidebar (5 Navigation-Items)
- Dashboard mit Status-Karten, Evidence-Summary, Attention-Queue
- Run-Detail-Ansicht mit Live-SSE-Events
- Repositories-Verwaltung
- Evidence-Explorer
- Settings-Seite
- Mode-Banner (Demo/Live)
- E2E-Testsuite mit 30+ Tests

### Live Operations (Issue #28, #29, #66)
- SSE Broadcaster mit Heartbeat, Secret-Redaction, Rate-Limiting
- Run Control: Pause, Abort, Resume, Retry
- Cancel-Endpoint mit Status-Protection
- Demo-Modus mit Live-Run-Seeding

### Sicherheit (Issue #44, #47, #50)
- Merge-Kill-Switch (default: off)
- Push-Policy (default: blocked)
- Agent Quarantine für externe Operatoren
- Secret-Redaction in SSE-Events
- CSP-konforme CORS-Konfiguration

### Qualitätssicherung
- 79 Unit/Integration-Tests (Vitest)
- 30+ E2E-Tests (Playwright)
- 15 UI-Qualitätstests (Accessibility, Layout, Navigation)
- Contract-Tests
- Visual Regression via Screenshots

## Installation

```bash
git clone https://github.com/xxammaxx/Positron.git
cd Positron
npm install
```

## Konfiguration

```bash
cp .env.example apps/server/.env
# Für echten GitHub-Zugriff:
# POSITRON_GITHUB_MODE=real
# GITHUB_TOKEN=ghp_...
# POSITRON_REPO_OWNER=dein_username
# POSITRON_REPO_NAME=dein_repo
```

## Bekannte Einschränkungen

1. **Sandbox immer Fake:** `RealGitWorkspaceAdapter` fehlt — Sandbox läuft nur im Fake-Modus
2. **Versionen:** Root: 3.0.0, Server: 0.1.0, Web: 3.0.0 — für finale Version harmonisieren
3. **Kein Dark/Light Theme:** Nur Dark-Mode
4. **Evidence Explorer:** Stub-Seite, noch nicht voll implementiert
5. **Settings-Seite:** Stub, MCP-Config noch nicht editierbar
6. **Mobile:** Sidebar-Collapse funktional aber CSS-Breakpoints fehlen
