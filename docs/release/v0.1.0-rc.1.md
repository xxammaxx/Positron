## Sicherheitsprofil (3 Stufen)

### Observe (Level 0)
- Nur lesender Zugriff auf GitHub (Issues, Labels, Comments)
- Keine Commits, kein Push, kein Merge
- Kein CLI-Zugriff auf SpecKit/OpenCode
- Ideal für: Monitoring, Analyse, Erstbewertung

### Supervised (Level 2) — **Empfohlen für Dogfood**
- GitHub Read+Write (Labels, Comments, PRs)
- Commits lokal erlaubt
- Push und Merge via `POSITRON_ENABLE_PUSH` + `POSITRON_ENABLE_MERGE` gegatet
- Kill-Switch blockiert sofort alle Merges
- Fix-Loop deaktiviert (optional aktivierbar)
- Reviewer-Zuweisung aktiv (konfiguriert via `POSITRON_PR_REVIEWERS`)
- SSE Live-Updates aktiv
- Run-Control (Pause/Abort) aktiv

### Autonomous-Safe (Level 3)
- Wie Supervised, plus:
- Push erlaubt (muss explizit aktiviert sein)
- Auto-Merge aktiv (muss explizit aktiviert sein)
- Fix-Loop aktiv (muss explizit aktiviert sein)
- Alle Safety Gates sichtbar im Dashboard

## Env-Variablen (Checkliste)

```bash
# === Pflicht ===
GITHUB_TOKEN=ghp_xxx                                    # GitHub Personal Access Token (repo scope)
POSITRON_REPO_OWNER=mein-owner                           # GitHub Owner
POSITRON_REPO_NAME=mein-repo                             # GitHub Repo Name

# === GitHub Modus ===
GITHUB_MODE=real                                         # real | fake (fake für Tests)

# === Push / Merge / Safety Gates ===
POSITRON_ENABLE_PUSH=false                               # true = Git Push erlauben
POSITRON_ENABLE_MERGE=false                              # true = Auto-Merge erlauben
POSITRON_MERGE_DRY_RUN=true                              # true = Merge simulieren
POSITRON_MERGE_KILL_SWITCH=false                         # true = ALLE Merges sofort blockieren

# === Fix-Loop ===
POSITRON_ENABLE_FIX_LOOP=false                           # true = Automatischer Retry
POSITRON_MAX_FIX_LOOPS=3                                 # Maximale Retry-Versuche

# === CLI-Adapter ===
POSITRON_SPECKIT_MODE=fake                               # real | fake
POSITRON_OPENCODE_MODE=fake                              # real | fake

# === Reviewer-Automation ===
POSITRON_PR_REVIEWERS=username1,username2                 # Komma-separierte GitHub Usernames
POSITRON_PR_TEAM_REVIEWERS=team-slug                     # Komma-separierte Team-Slugs
```

## Release-Checkliste

### Vor dem ersten Dogfood-Run
- [ ] `.env` vollständig konfiguriert
- [ ] Safety Gates auf `supervised` gesetzt (Push+Merge OFF)
- [ ] GitHub Token mit `repo` Scope hinterlegt
- [ ] Positron läuft lokal (`npm run build && cd apps/server && npx tsx src/index.ts`)
- [ ] Web UI erreichbar (http://localhost:3000)
- [ ] Playwright E2E-Tests grün (`npx playwright test --config apps/web/playwright.config.ts`)
- [ ] Backup von `.positron/` und SQLite existiert (falls vorhanden)

### Während des Dogfood-Runs
- [ ] Dashboard überwachen (Run-Status, Merge-Gates, Safety Controls)
- [ ] Bei Kill-Switch-Auslösung: Ursache dokumentieren
- [ ] Bei Fehlern: Run-Controls (Pause/Abort) nutzen
- [ ] Alle unerwarteten Events im GitHub Issue dokumentieren

### Nach dem Dogfood-Run
- [ ] Ergebnis dokumentieren (Erfolg/Fehler/Erkenntnisse)
- [ ] Bei Erfolg: Safety Gates schrittweise erhöhen
- [ ] Bei Fehlern: Issue aufmachen, Fix vor nächstem Run

## Bekannte Einschränkungen (Known Issues)

1. **Synchrone Runs** — POST /api/runs blockiert bis zur Completion (kein Background-Processing)
2. **In-Memory-Daten** — Nach Server-Neustart sind alle Runs weg (SQLite-Persistenz für Runs geplant)
3. **Keine Authentifizierung** — Dashboard ohne Login erreichbar
4. **Kein Docker-Container** — Läuft aktuell nur nativ
5. **Mid-Phase-Abort nicht möglich** — Abort wirkt nur zwischen Phasen

## Entscheidung

**Ready for dogfood on real repo: JA** ✅

Unter folgenden Bedingungen:
- Supervised-Mode (Push+Merge OFF, Kill-Switch bereit)
- GitHub Token mit minimalen Rechten
- Persönliches Test-Repo (nicht Production)
- Beobachtung durch Operator während des Runs
