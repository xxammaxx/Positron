# Vibe Coding Context Manifest Template

> Version: 1.0.0 | Status: PROPOSED | Date: 2026-06-10
> Usage: Generated by Positron orchestrator before each agent run

---

## Purpose

Das Context Manifest ist die strukturierte Brücke zwischen Positrons Orchestrator und dem Coding-Agenten. Es enthält ALLE Informationen, die der Agent für seine Arbeit benötigt — nicht mehr, nicht weniger. Der Agent bekommt nicht den gesamten Workspace, sondern ein kuratiertes Manifest.

---

## Field Descriptions

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `run.id` | UUID | Yes | Eindeutige Run-ID |
| `run.phase` | string | Yes | Aktuelle Pipeline-Phase |
| `run.autonomyLevel` | 0-4 | Yes | Autonomie-Level des Runs |
| `run.attempt` | number | Yes | Aktueller Versuch (1-based) |
| `issue.number` | number | Yes | GitHub Issue-Nummer |
| `issue.body` | string | Yes | Vollständiger Issue-Text |
| `workspace.path` | string | Yes | Pfad zum isolierten Workspace |
| `specification.artifacts` | object | No | Pfade zu Spec-Artefakten |
| `verificationContract.acceptanceCriteria` | string[] | Yes | Prüfbare Kriterien |
| `redTests.paths` | string[] | No | Pfade zu Red-Test-Dateien |
| `context.affectedModules` | string[] | Yes | Module, die geändert werden sollen |
| `agent.type` | string | Yes | Agent-Typ (opencode, codex, claude-code, ...) |
| `constraints.constitution` | string | Yes | Pfad zur Constitution-Datei |
| `evidenceRequirements` | object | Yes | Welche Evidence muss gesammelt werden |

---

## Generation Rules

1. **Minimal:** Nur Felder ausfüllen, die für die aktuelle Phase relevant sind
2. **Verifiziert:** Keine Halluzinationen — nur belegbare Werte
3. **Redigiert:** Secrets und Tokens werden vor der Erzeugung redigiert
4. **Versioniert:** Jedes Manifest hat eine Schema-Version
5. **Attribuiert:** `generatedBy` und `generatedAt` sind immer gesetzt

---

## Agenten-Konsum

Der Agent erhält das Manifest als:

- **CLI-Agenten:** JSON-Datei im Workspace (`{workspace}/.positron/context-manifest.json`)
- **API-Agenten:** Als strukturierter Prompt-Teil
- **MCP-Agenten:** Über MCP-Tool-Aufruf mit Manifest-Content
