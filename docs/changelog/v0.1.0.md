# Positron v0.1.0 — Erster stabiler Release

**Datum:** 2026-05-27  
**Commit:** 4d3a5af

## Überblick

Erster stabiler Release von Positron — dem Evidence-Gated GitHub Issue Execution System.  
Nach 79 Issues, ~40 Feature-Branches und 124 Tests ist die Plattform operationell nutzbar.

## Features

### Pipeline (Issues #1–#13, #14–#27, #28–#62)
- Vollständige 27-Phasen-Pipeline: QUEUED → CLAIMED → SPECIFY → ... → DONE/FAILED
- Speckit-Workflow: Spec → Plan → Tasks → Analyze → Review → Implement → Test → Verify
- Fix-Loop mit Expo-Backoff (max 3 Wiederholungen)
- Evidence-Gates: Jede Phase produziert prüfbare Artefakte
- Run-State-Machine mit Resumption aus Events

### GitHub Integration
- GitHub Adapter (Fake/Real Strategy Pattern)
- Issue-Listen, PR-Erstellung, Merge, Labels, Review-Request
- Auto-Merge mit Dry-Run, Kill-Switch, Mergeability-Prüfung
- GitHub Watcher für automatische Run-Erstellung

### Web UI
- App Shell mit Sidebar (5 Navigation-Items)
- Dashboard mit Status-Karten, Evidence-Summary, Attention-Queue, Recent-Activity
- Run-Detail-Ansicht mit Live-SSE-Events + Live-Evidence-Panel
- Repositories-Verwaltung
- Evidence-Explorer mit Filter, Suche, Summary-Cards
- Settings-Seite mit Safety-Gates, MCP-Server-Config, Test-Modes
- Mode-Banner (Demo/Live)
- Theme-Support (Dark/Light Mode)
- Mobile Responsive Design (Sidebar-Overlay, Breakpoints)

### Live Operations (Issues #28, #29, #66)
- SSE Broadcaster mit Heartbeat, Secret-Redaction (9 Pattern-Typen), Rate-Limiting (20 evt/s)
- Sequence-Numbers + W3C Last-Event-ID für Reconnection
- Run Control: Pause, Abort, Resume, Retry
- Cancel-Endpoint (idempotent, status-protected, atomic DB update)
- Demo-Modus mit Live-Run-Seeding + Events + Evidence

### Sicherheit
- Merge-Kill-Switch (default: off)
- Push-Policy (default: blocked)
- Agent Quarantine für externe Operatoren
- Secret-Redaction in SSE-Events (GitHub-Tokens, Slack, AWS, Bearer, etc.)
- CSP-konforme CORS-Konfiguration
- Data Audit Dokumentation (docs/ui-audit/)

### Qualitätssicherung
- 66 Unit/Integration-Tests (Vitest)
- 58 Web Frontend-Tests (Vitest + React Testing Library)
- 30+ E2E-Tests (Playwright)
- 15 UI-Qualitätstests (Accessibility, Layout, Navigation)
- Visual Regression via Screenshots
- Automatisierte Issue-Verifikation (scripts/verify-issues.mjs)

## Installation

```bash
git clone https://github.com/xxammaxx/Positron.git
cd Positron
npm install
# Für echten GitHub-Zugriff:
cp .env.example apps/server/.env
```

## Tests

```bash
npm test                    # Vitest (66 Tests)
npm run test:e2e            # Playwright headless
npm run test:e2e:headed     # Playwright sichtbar
```

## Konfiguration

```
POSITRON_GITHUB_MODE       = fake|real            (default: fake)
POSITRON_WORKSPACE_ROOT    = /pfad/zum/workspace   (optional)
POSITRON_ENABLE_PUSH       = true                  (default: false)
POSITRON_ENABLE_MERGE      = true                  (default: false)
POSITRON_MERGE_KILL_SWITCH = true                  (default: true)
POSITRON_ENABLE_FIX_LOOP   = true                  (default: false)
```
