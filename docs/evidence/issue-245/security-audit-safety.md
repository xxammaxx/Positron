## Audit Entry Contents (by callback implementer)

The `onAudit` callback implementer should log:
- `runId` from `call.runId`
- `toolId` from `call.toolId`
- `timestamp` (generated by callback)
- `decision`: "pre_execution_audit"
- `reason`: "audit gate passed"

**Security constraints for callback implementer:**
- Must NOT log raw `call.arguments` without sanitization
- Must NOT log secrets, tokens, or .env data
- If sanitization unavailable, log only metadata (toolId, runId, timestamp)
