## Recommended Actions

1. **Wait for server startup:** Check `curl -s http://localhost:9000/api/system/status` until status is "UP"
2. **Login to SonarQube:** http://localhost:9000 (default: admin/admin, change password on first login)
3. **Generate token:** My Account → Security → Generate Token → Save as `SONAR_TOKEN`
4. **Run test analysis:**
   ```bash
   export SONAR_HOST_URL=http://localhost:9000
   export SONAR_TOKEN=<your-token>
   npm test  # generates coverage/lcov.info
   npx sonarqube-scanner
   node scripts/sonarqube-quality-gate.mjs
   ```
5. **Configure CI secrets:**
   - `SONAR_HOST_URL` → SonarQube instance URL (must be CI-accessible)
   - `SONAR_TOKEN` → Analysis token
6. **Note:** CI SonarQube requires a CI-accessible instance (not localhost). Options: cloud-hosted SonarQube, CI runner with Docker, or skip CI SonarQube and use local-only.
